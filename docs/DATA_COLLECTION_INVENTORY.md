# 데이터 수집 인벤토리\n\n## 개요\n\n설화 정비소는 다음과 같은 데이터를 수집/저장/전송합니다. **개인식별정보(PII)는 수집하지 않습니다.**\n\n---\n\n## 1. 로컬 저장 데이터 (기기 내)\n\n### 게임 진행 데이터\n- **스테이지 진행**: 현재 스테이지, 클리어 여부\n- **재화**: 골드, 파편, 코스튬 토큰, 패스 토큰\n- **강화/업그레이드**: 각 업그레이드 레벨\n- **동료**: 해금 여부, 강화 레벨, 커스터마이징(닉네임, 오라, 색상)\n- **정비소 데코**: 보유 데코, 배치, 프리셋\n- **칭호/뱃지**: 해금 여부\n- **업적**: 진행도, 완료 여부\n- **일일 미션**: 진행도, 완료 여부\n- **패스**: 레벨, 보상 수령 여부\n- **UGC 시나리오**: 생성/수정한 정비 의뢰 데이터\n- **친구/정비반**: 친구 코드, 정비반 멤버 목록(익명)\n- **랭킹**: 주간 랭킹 점수\n\n### 설정 데이터\n- **언어**: 선택한 언어(ko/en)\n- **음소거/음량**: 음악/효과음 설정\n- **알림**: 알림 활성화 여부\n- **접근성**: 글자 크기, 고대비, 모션 감소\n- **데이터 사용**: 오프라인 모드, 네트워크 설정\n\n### 기술 데이터 (로컬)\n- **기기 ID**: 익명 UUID (로컬 생성, 서버 미전송)\n- **앱 버전**: 설치된 앱 버전\n- **마지막 플레이 시간**: 오프라인 보상 계산용\n- **클라우드 백업 코드**: Base64 인코딩 (로컬 저장)\n\n---\n\n## 2. 원격 전송 데이터 (서버/클라우드)\n\n### 분석 이벤트 (선택적, Mock 구현)\n- **이벤트 타입**: battle_start, battle_win, upgrade_purchase, stage_clear, etc.\n- **타임스탬프**: 이벤트 발생 시간\n- **세션 ID**: 익명 세션 식별자\n- **기기 정보**: OS, 앱 버전 (PII 없음)\n- **게임 상태 스냅샷**: 현재 스테이지, 재화 (집계용)\n\n### 크래시 리포트 (선택적, Mock 구현)\n- **에러 메시지**: 스택 트레이스\n- **타임스탬프**: 크래시 발생 시간\n- **기기 정보**: OS, 앱 버전\n- **앱 상태**: 메모리, 배터리 (PII 없음)\n\n### 클라우드 세이브 (선택적)\n- **저장 데이터**: 위의 \"게임 진행 데이터\" 전체\n- **백업 코드**: 기기 변경 시 복원용\n- **타임스탬프**: 마지막 동기화 시간\n\n### 광고/결제 이벤트 (선택적, Mock 구현)\n- **광고 시청**: 광고 유형, 시청 완료 여부\n- **결제**: SKU, 금액, 통화 (PII 없음)\n- **구매 복원**: 이전 구매 이력 (로컬)\n\n---\n\n## 3. 수집하지 않는 데이터 (PII)\n\n❌ **이름, 이메일, 전화번호, 주소, 생년월일**\n❌ **IP 주소, MAC 주소, IMEI**\n❌ **위치 정보(GPS)**\n❌ **연락처, 캘린더, 파일 시스템**\n❌ **카메라, 마이크 (권한 요청 없음)**\n\n---\n\n## 4. 권한 요청\n\n### Android\n- **알림 (POST_NOTIFICATIONS)**: 선택적, 사용자 동의 후\n- **인터넷**: 필수 (클라우드 세이브, 광고, 분석)\n\n### iOS (옵션)\n- **알림 (UNUserNotificationCenter)**: 선택적\n- **인터넷**: 필수\n- **ATT (App Tracking Transparency)**: 광고 추적 동의 (선택적)\n\n---\n\n## 5. 데이터 삭제 경로\n\n### 로컬 데이터 삭제\n- **Settings → 데이터 초기화 → 확인**\n- 모든 게임 진행 데이터, 설정, 로컬 저장소 삭제\n- 앱 재시작 시 새 게임 시작\n\n### 클라우드 백업 삭제\n- **Settings → 클라우드 → 백업 삭제**\n- 클라우드 세이브 서버에서 데이터 삭제 (구현 시)\n- 로컬 백업 코드는 유효하지 않게 됨\n\n### 계정 삭제 (향후)\n- 개인 계정 시스템 도입 시, 계정 삭제 옵션 추가\n- 모든 클라우드 데이터 삭제\n\n---\n\n## 6. 데이터 보안\n\n- **로컬 저장**: MMKV 또는 AsyncStorage (암호화 옵션)\n- **전송**: HTTPS (TLS 1.2+)\n- **서버**: 표준 보안 관행 (암호화, 접근 제어)\n- **보관 기간**: 분석 데이터 90일, 크래시 리포트 30일\n\n---\n\n## 7. 제3자 서비스\n\n### 광고 네트워크 (선택적)\n- **Google Mobile Ads**: 광고 노출, 클릭 (PII 없음)\n- **Facebook Audience Network**: 광고 노출 (PII 없음)\n\n### 결제 처리\n- **Google Play Billing**: 결제 정보 (PII 없음, 스토어에서 처리)\n- **Apple In-App Purchase**: 결제 정보 (PII 없음, 스토어에서 처리)\n\n### 분석 (선택적)\n- **Firebase Analytics**: 이벤트 분석 (PII 없음)\n- **Sentry**: 크래시 리포팅 (PII 없음)\n\n---\n\n## 8. 사용자 권리\n\n- **접근권**: Settings에서 저장된 모든 데이터 확인 가능\n- **수정권**: 게임 진행 데이터 직접 수정 불가 (게임 플레이를 통해서만)\n- **삭제권**: 위의 \"데이터 삭제 경로\" 참고\n- **이동권**: 클라우드 백업 코드로 다른 기기로 이동 가능\n- **거부권**: 분석/광고 추적 거부 가능 (Settings)\n\n---\n\n## 9. 문의\n\n- **개인정보 관련 문의**: support@example.com\n- **데이터 삭제 요청**: support@example.com\n- **버그/보안 보고**: security@example.com\n\n---\n\n**마지막
